## 背景・目的

Flutterアプリを廃止しWebアプリ（golf-web）に一本化する移行にあたり、旧設計の残骸（`plays/`系）を削除し、`rounds/`系を中心とした整理されたナビゲーション構造に再編する。合わせてダッシュボードをトップページとして新設する。

- `app/plays/` 配下すべて
- `src/lib/apiClient.ts`（モックAPI）

## 共通レイアウト

`app/layout.tsx` に共通ヘッダーナビを追加する。

**ヘッダー構成**:
```
[ Golf Stuff ]  |  ダッシュボード  ラウンド  ゴルフ場
```

- 左：サイトタイトル "Golf Stuff"（`/` へのリンク）、縦の区切り線
- ナビリンク：ダッシュボード・ラウンド・ゴルフ場
- アクティブ状態：現在パスに対応するリンクをボトムボーダーでハイライト（`usePathname` 使用）
- PC前提。モバイル対応は対象外。

## ダッシュボード（`/`）

### 表示ブロック

**1. 生涯サマリー**
- ベストスコア（合計スコア最小値）：コース名・日付付き
- ベストハーフ（9Hスコア最小値）：コース名・ハーフ名・日付付き

**2. 今年サマリー**
- ラウンド数
- 今年のベストスコア
- 平均スコア（小数第1位）

**3. スコア推移グラフ**
- ボタンで2モード切り替え：
  - **年別平均**：年ごとの平均スコアを棒グラフ
  - **直近20R**：直近20ラウンドのスコアを棒グラフ（左=古、右=新）
- グラフライブラリ：Recharts
- `/rounds` へのリンクを「すべて見る」として配置

**4. 直近ラウンド一覧**
- 最新5件（日付・コース名・合計スコア）
- `/rounds` へのリンクを「すべて見る」として配置

### データ取得

Server Componentとして実装し、Prismaで直接DBから取得する。

```ts
// 必要なクエリ
prisma.trnRoundHoleResult.aggregate(...)  // ベストスコア・平均
prisma.trnRound.findMany(...)             // ラウンド一覧・直近
```

ハーフスコアはラウンドのホール結果から `outin` や `holeNumber` で集計する（現スキーマはハーフを直接持たないため、9穴ごとに集計）。

## デザイン方針

- **スタイリング**：Tailwind CSS（既存設定済み）
- **シグネチャ要素**：スコア数値を大きく（40px）表示、装飾なし。紙のスコアカードの「数字が主役」という精密さを参照
- **カラー**：インタラクティブ要素のみアクセントカラー、スコア数値はモノクロ（緑ベースのゴルフらしいデザインを意図的に避ける）
- **ボーダー**：0.5px の細い区切り線で構造を作る
- **角丸**：カードは 12px
